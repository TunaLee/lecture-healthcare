Bagging vs Boosting: 근본적 차이

핵심 차이

  • Bagging: "여러 명이 동시에 문제를 풀고 답을 평균"
  • Boosting: "첫 번째 사람이 틀린 문제를 두 번째 사람이 집중해서 풀기"

Boosting의 직관적 이해

학습 과정의 비유: 수학 문제집 풀기

  1. 1단계: 처음엔 쉬운 문제부터 풀기 (약한 학습기 1)
  2. 2단계: 틀린 문제 표시하고 다시 풀기 (약한 학습기 2)
  3. 3단계: 또 틀린 문제에 더 집중하기 (약한 학습기 3)
  4. 반복: 계속해서 틀린 부분을 보완

→ 결국 모든 문제를 잘 풀 수 있게 됨!

약한 학습기 (Weak Learner)

정의: 랜덤 추측보다 약간 나은 성능의 모델

  • 보통 깊이가 1~3인 Decision Tree (Decision Stump)
  • 정확도가 50%를 약간 넘는 수준
  • 단순하고 빠른 모델

강한 학습기로의 변신

여러 약한 학습기를 순차적으로 결합하면:

  • 각 단계에서 이전 단계의 실수를 학습
  • 어려운 샘플에 점점 더 집중
  • 최종적으로 매우 강력한 모델 완성

AdaBoost (Adaptive Boosting)

AdaBoost = Adaptive + Boosting

1995년 Freund & Schapire가 제안한 최초의 실용적인 Boosting 알고리즘

"적응적으로" 어려운 샘플에 가중치를 높여가며 학습

핵심 아이디어

  1. 초기: 모든 샘플에 동일한 가중치
  2. 학습: 첫 번째 약한 학습기 훈련
  3. 평가: 잘못 분류된 샘플 찾기
  4. 가중치 업데이트: 틀린 샘플의 가중치 증가, 맞은 샘플의 가중치 감소
  5. 반복: 업데이트된 가중치로 다음 학습기 훈련

왜 효과적인가?

어려운 샘플에 집중함으로써 전체적인 성능 향상

각 학습기가 이전 학습기의 약점을 보완

AdaBoost: Weak Learner에서 Strong Learner로

핵심 아이디어

Weak Learner(약한 학습기)란?

  • 정의: 랜덤보다는 조금 나은 수준의 분류기 (정확도 50% 이상)
  • 예시: Decision Stump (깊이가 1인 결정 트리)
  • 특징: 단순하고 빠르지만, 성능이 낮음

Strong Learner(강한 학습기)란?

  • 정의: 높은 정확도를 가진 분류기
  • AdaBoost의 목표: 여러 Weak Learner를 결합하여 Strong Learner 생성
  • 핵심 철학: "세 사람이 모이면 문수보살의 지혜" - 집단 지성

순차적 학습 과정

  1. 1차 모델 학습
    • 모든 데이터를 동일한 가중치로 시작
    • 첫 번째 Weak Learner 학습
    • 틀린 데이터의 가중치 증가
  2. 2차 모델 학습
    • 1차에서 틀린 데이터에 집중
    • 새로운 Weak Learner 학습
    • 다시 틀린 데이터의 가중치 증가
  3. 반복
    • 정해진 횟수만큼 반복
    • 각 모델은 이전 모델이 틀린 부분에 집중
  4. 최종 결합
    • 모든 Weak Learner를 가중 투표로 결합
    • Strong Learner 완성

비유로 이해하기

시험 공부 과정과 유사:

  • 1차 모의고사: 전체적으로 공부 → 틀린 문제 파악
  • 2차 모의고사: 틀린 유형 집중 공부 → 또 다른 약점 발견
  • 3차 모의고사: 새로운 약점 보완 → 점점 개선
  • 최종 시험: 모든 학습 결과를 종합하여 높은 점수

Boosting의 두 가지 핵심 요소

1️⃣ 데이터 가중치 (Data Weights)

왜 중요한가?

다음 모델에게 "어떤 데이터를 중요하게 봐야 하는지" 알려주는 역할

작동 방식

  • 처음: 모든 데이터의 가중치 동일 (예: 각 1/N)
  • 학습 후: 틀린 데이터의 가중치 증가, 맞은 데이터의 가중치 감소
  • 다음 모델: 가중치가 큰 데이터에 더 집중
  • 효과: 어려운 데이터를 점점 더 잘 분류

예시

1차 모델: 데이터 A, B, C 중 B를 틀림

→ B의 가중치 증가 (0.1 → 0.3)

2차 모델: B에 3배 더 집중하여 학습

→ B를 맞추는 데 더 노력

핵심 포인트

가중치 = 다음 모델에게 보내는 "이 데이터 중요해요!" 메시지

이를 통해 순차적으로 학습하면서 점점 어려운 케이스를 학습

2️⃣ 모델 신뢰도 (Model Confidence)

신뢰도란?

정의

각 Weak Learner가 얼마나 믿을 만한지를 나타내는 점수

신뢰도 (α) = ½ ln((1-ε)/ε)

ε = 에러율 (틀린 비율)

신뢰도의 특성

  • 에러율 낮음 (정확함): 신뢰도 높음 (α 큼)
  • 에러율 높음 (부정확함): 신뢰도 낮음 (α 작음)
  • 에러율 50% (랜덤): 신뢰도 0 (무시)

최종 결과에 영향력 부여

신뢰도 기반 가중 투표

핵심: 정확한 모델의 의견을 더 중요하게 반영

예시:

모델 1 (에러율 0.4) → 신뢰도 0.203

모델 2 (에러율 0.2) → 신뢰도 0.745

최종 예측 = 0.203 × (모델1 예측) + 0.745 × (모델2 예측)

→ 모델 2의 의견이 약 3.7배 더 중요하게 반영됨

왜 중요한가?

민주주의 투표와 유사:

  • 전문가의 의견(정확한 모델)에 더 큰 가중치
  • 비전문가의 의견(부정확한 모델)에 작은 가중치
  • 결과: 집단 지성의 최적화

신뢰도를 바탕으로 데이터 가중치 변경

가중치 업데이트 공식

맞은 데이터: 새 가중치 = 현재 가중치 × e^(-α)

틀린 데이터: 새 가중치 = 현재 가중치 × e^(α)

α: 모델의 신뢰도

  • 신뢰도가 높을수록 → 가중치 변화가 큼
  • 신뢰도가 낮을수록 → 가중치 변화가 작음

구체적 예시 (이미지 참고)

1차 모델 학습

초기 상태: 5개 데이터, 각 가중치 0.2

1차 모델 예측: 2개 틀림, 3개 맞음

에러율: ε = 0.4 (40%)

신뢰도 계산: α = ½ln(0.6/0.4) = 0.203

변화율: e^0.203 = 1.22

가중치 업데이트

맞은 데이터 (3개):

0.2 × 1 = 0.2 → 정규화 후 0.184

틀린 데이터 (2개):

0.2 × 1.22 = 0.244 → 정규화 후 0.224

2차 모델 학습

이전 모델의 신뢰도를 바탕으로 학습

새로운 가중치로 학습: 1차에서 틀린 데이터(0.224)에 더 집중

2차 예측: 다른 데이터 1개 틀림

에러율: ε = 0.184 (가중치 합)

신뢰도: α = ½ln(0.816/0.184) = 0.745

→ 2차 모델이 1차보다 더 정확하므로 신뢰도가 3.7배 높음!

신뢰도 기반 최종 예측

추론 하는 과정에서는

학습 데이터의 가중치는 필요 없음!

추론 시에는 모델의 신뢰도를 이용해 결과에 적용

최종 예측 공식

최종 결과 = sign(Σ α_i × h_i(x))

α_i = i번째 모델의 신뢰도
h_i(x) = i번째 모델의 예측 (-1 또는 +1)

구체적 예시

상황 설정 (이미지 3 참고)

  • 모델 1: 신뢰도 0.203, 예측 = 0.6 (클래스 0 선호)
  • 모델 2: 신뢰도 0.745, 예측 = 0.8 (클래스 1 선호)

가중 투표 계산

각 모델의 기여도:

모델 1의 기여: 0.6 × 0.203 = 0.122

모델 2의 기여: 0.8 × 0.745 = 0.596

최종 점수:

0.122 + 0.596 = 0.718

→ 0.718 > 0.5 이므로 클래스 1로 예측!

해석: 모델 2가 더 믿을 만하므로(신뢰도 0.745 > 0.203), 모델 2의 의견(클래스 1)이 최종 결정에 더 큰 영향을 미침

핵심 정리

  1. 학습 시: 데이터 가중치로 다음 모델 학습 방향 결정
  2. 추론 시: 모델 신뢰도로 최종 예측 결정
  3. 결과: 정확한 모델의 의견이 더 중요하게 반영됨

AdaBoost 알고리즘 시각화

Boosting: 반복적 학습 과정

단계별 설명

🔵 Iteration 1 - 첫 번째 약한 학습기

  • 초기 상태: 모든 샘플에 동일한 가중치 부여
  • 분류 경계: 세로선(vertical line)으로 데이터를 나눔
  • 결과: 왼쪽에 + (파란색), 오른쪽에 - (주황색) 배치
  • 문제점: 일부 샘플들이 잘못 분류됨 (주황 영역에 + 샘플, 파란 영역에 - 샘플)

🟠 Iteration 2 - 두 번째 약한 학습기

  • 가중치 조정: Iteration 1에서 틀린 샘플들의 가중치를 증가
  • 분류 경계: 이전과 다른 위치의 세로선으로 분류
  • 목표: 이전에 틀린 샘플들을 올바르게 분류하는 데 집중
  • 개선: 왼쪽 파란색 영역이 확장되어 더 많은 + 샘플 포함

🟡 Iteration 3 - 세 번째 약한 학습기

  • 추가 조정: Iteration 2에서도 틀린 샘플들에 다시 집중
  • 분류 경계: 가로선(horizontal line)으로 변경
  • 전략: 위쪽에 + (파란색), 아래쪽에 - (주황색) 배치
  • 목적: 이전 두 모델이 놓친 패턴을 보완

🎯 Final Classifier / Strong Classifier

  • 결합 방법: 3개의 약한 학습기를 가중 평균으로 결합
  • 복잡한 경계: 수직선과 수평선이 조합되어 격자 모양의 복잡한 결정 경계 생성
  • 성능 향상: 각 학습기가 서로 다른 영역의 오류를 보완하여 전체 정확도 향상
  • 최종 예측: Final = α₁ × h₁(x) + α₂ × h₂(x) + α₃ × h₃(x)

💡 핵심 인사이트

순차적 오류 보완: 각 반복(iteration)마다 이전 모델이 틀린 샘플에 가중치를 높여, 다음 모델이 그 샘플들을 더 잘 분류하도록 유도합니다.

단순한 모델의 힘: 각 약한 학습기는 단순한 선 하나로만 분류하지만, 여러 개를 결합하면 복잡한 패턴도 학습 가능합니다.

다양성: 각 학습기가 서로 다른 방향(세로, 가로)의 경계를 만들어 다양한 관점에서 데이터를 분석합니다.

AdaBoost 알고리즘 상세

알고리즘 단계

1. 초기화

모든 샘플의 가중치: w₁(i) = 1/N (N: 샘플 수)

2. T번 반복 (t = 1, 2, ..., T)

  1. 학습: 가중치 wₜ를 사용해 약한 분류기 hₜ 훈련
  2. 오차 계산: εₜ = Σ wₜ(i) × I(yᵢ ≠ hₜ(xᵢ))
  3. 분류기 중요도: αₜ = 0.5 × log((1-εₜ)/εₜ)
  4. 가중치 업데이트:
    • 맞은 경우: wₜ₊₁(i) = wₜ(i) × e^(-αₜ)
    • 틀린 경우: wₜ₊₁(i) = wₜ(i) × e^(αₜ)
  5. 정규화: wₜ₊₁ = wₜ₊₁ / Σwₜ₊₁

3. 최종 예측

H(x) = sign(Σ αₜ × hₜ(x))

직관적 이해

  • εₜ가 작을수록 (정확할수록): αₜ가 커짐 → 더 큰 영향력
  • 틀린 샘플: e^αₜ만큼 가중치 증가
  • 맞은 샘플: e^(-αₜ)만큼 가중치 감소

모델 평가 지표: 정확도를 넘어서

왜 정확도만으로는 부족한가?

불균형 데이터 문제

암 진단: 전체 환자의 2%만 실제 암 → 모든 환자를 "정상"으로 예측해도 98% 정확도!

주요 평가 지표

1. Confusion Matrix (혼동 행렬)

예측: Positive 예측: Negative
실제: Positive True Positive (TP) False Negative (FN)
실제: Negative False Positive (FP) True Negative (TN)

2. Precision (정밀도)

Precision = TP / (TP + FP)

"Positive로 예측한 것 중 실제 Positive 비율"

예: 스팸 필터 - 스팸으로 분류한 메일 중 진짜 스팸 비율

3. Recall (재현율) = Sensitivity

Recall = TP / (TP + FN)

"실제 Positive 중 Positive로 예측한 비율"

예: 암 진단 - 실제 암 환자 중 암으로 진단한 비율

4. F1 Score

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Precision과 Recall의 조화평균

Trade-off

  • Precision ↑ → Recall ↓: 확실한 경우만 Positive 예측
  • Recall ↑ → Precision ↓: 놓치지 않으려고 많이 Positive 예측

ROC-AUC와 Precision-Recall Curve

ROC Curve (Receiver Operating Characteristic)

  • X축: False Positive Rate (FPR) = FP / (FP + TN)
  • Y축: True Positive Rate (TPR) = Recall
  • AUC: Area Under Curve (0.5~1.0, 높을수록 좋음)

ROC-AUC 해석

  • AUC = 0.5: 랜덤 추측 수준
  • AUC = 0.7~0.8: 적절한 성능
  • AUC = 0.8~0.9: 우수한 성능
  • AUC > 0.9: 매우 우수한 성능

Precision-Recall Curve

  • X축: Recall
  • Y축: Precision
  • AP: Average Precision (면적)

언제 어떤 것을 사용할까?

  • ROC-AUC: 클래스 균형이 맞을 때, 전반적 성능
  • PR Curve: 불균형 데이터, Positive 클래스가 중요할 때
  • 의료 진단: PR Curve 중요 (암 환자가 매우 적음)

Classification Report

Gradient Boosting: 더 강력한 부스팅

AdaBoost의 한계

  • 이진 분류에 최적화
  • 노이즈와 이상치에 민감
  • 확장성 제한

Gradient Boosting의 혁신

  • 일반화: 모든 미분 가능한 손실 함수 사용 가능
  • 유연성: 분류, 회귀, 랭킹 등 다양한 문제 해결
  • 최적화: 경사하강법 원리 적용

핵심 아이디어: 잔차(Residual) 학습

  1. 첫 번째 모델로 예측
  2. 실제값과 예측값의 차이(잔차) 계산
  3. 잔차를 타겟으로 두 번째 모델 학습
  4. 반복: 계속 잔차를 줄여나감

Gradient Boosting: 잔차 학습 과정

Gradient Boosting 수학적 이해

손실 함수 최소화

목표: argmin Σ L(yᵢ, F(xᵢ))

L: 손실 함수 (MSE, Log Loss 등)
F: 최종 예측 함수

경사하강법 연결

핵심 통찰: 잔차는 손실 함수의 음의 기울기!

  • 회귀 (MSE): 잔차 = yᵢ - F(xᵢ)
  • 이는 -∂L/∂F와 같음
  • 즉, 잔차를 학습하는 것 = 경사하강법

알고리즘

  1. 초기화: F₀(x) = argmin Σ L(yᵢ, c)
  2. m = 1 to M 반복:
    • 의사 잔차 계산: rᵢₘ = -[∂L(yᵢ, F(xᵢ))/∂F(xᵢ)]
    • rᵢₘ을 타겟으로 약한 학습기 hₘ 학습
    • 최적 스텝 크기 γₘ 계산
    • 모델 업데이트: Fₘ(x) = Fₘ₋₁(x) + η × γₘ × hₘ(x)

XGBoost: 더 빠르고 더 정확!

XGBoost의 추가 똑똑한 기능들

1. 정규화 (Regularization)

문제: 100번째 모델이 학생 A에게 딱 맞춰지면?

→ 과적합! 새로운 학생 예측 못함

XGBoost 해결책:

목적함수 = 오차 + γ×(트리 복잡도) + λ×(가중치 크기)
  • γ (gamma): 트리가 너무 복잡하면 페널티
  • λ (lambda): 예측값이 너무 극단적이면 페널티

"정확하되, 너무 복잡하지 않게!"

2. 결측치 자동 처리

상황: 학생 E의 공부 시간 데이터 없음

GBM: 에러 발생 또는 전처리 필요

XGBoost:

  1. 결측치를 왼쪽으로 보냈을 때 점수 계산
  2. 결측치를 오른쪽으로 보냈을 때 점수 계산
  3. 더 좋은 쪽 선택!

데이터 전처리 시간 절약!

3. 병렬 처리

예시: 10개의 특성(공부시간, 출석률, 과제점수...)

  • GBM: 특성 1 확인 → 특성 2 확인 → ... (순차)
  • XGBoost: 10개 특성 동시 확인! (병렬)

결과: 특성이 많을수록 속도 차이가 극명!

GBM vs XGBoost 순차적 학습 핵심 정리

GBM 순차적 학습

  1. 초기 예측 (평균 등)
  2. 오차 계산 (실제 - 예측)
  3. 오차를 예측하는 트리 학습
  4. 예측 업데이트 (+ 학습률 × 새 예측)

  2. 1차 + 2차 미분 계산
  3. 최적값 = -g/h 계산
  4. 정규화 적용하여 트리 학습
  5. 예측 업데이트 (더 똑똑하게)
  6. 2~5 반복 (병렬로)

핵심: 오차를 "최적으로" 학습

비유로 완전 이해하기

# 회귀 데이터 생성
from sklearn.ensemble import GradientBoostingRegressor

# 비선형 데이터 생성
np.random.seed(42)
X_reg = np.sort(5 * np.random.rand(100, 1), axis=0)
y_reg = np.sin(X_reg).ravel() + np.random.normal(0, 0.1, X_reg.shape[0])

# 다양한 n_estimators로 학습
n_estimators_list = [1, 10, 50, 100]
fig, axes = plt.subplots(2, 2, figsize=(12, 10))

X_test_reg = np.arange(0.0, 5.0, 0.01)[:, np.newaxis]

for idx, n_est in enumerate(n_estimators_list):
    ax = axes[idx // 2, idx % 2]
    
    # 모델 학습
    gb_reg = GradientBoostingRegressor(
        n_estimators=n_est,
        learning_rate=0.1,
        max_depth=3,
        random_state=42
    )
    gb_reg.fit(X_reg, y_reg)
    
    # 예측
    y_pred = gb_reg.predict(X_test_reg)
    
    # 시각화
    ax.scatter(X_reg, y_reg, c='blue', s=20, alpha=0.5, label='Training data')
    ax.plot(X_test_reg, y_pred, 'r-', lw=2, label='Prediction')
    ax.plot(X_test_reg, np.sin(X_test_reg), 'g--', lw=1, 
            alpha=0.5, label='True function')
    
    ax.set_title(f'n_estimators = {n_est}')
    ax.set_xlabel('X')
    ax.set_ylabel('y')
    ax.legend()
    
    # MSE 표시
    mse = np.mean((gb_reg.predict(X_reg) - y_reg) ** 2)
    ax.text(0.02, 0.98, f'MSE: {mse:.4f}', 
            transform=ax.transAxes, fontsize=11,
            verticalalignment='top',
            bbox=dict(boxstyle='round', facecolor='white', alpha=0.8))

plt.suptitle('Gradient Boosting Regressor: 부스팅 라운드별 성능', fontsize=14)
plt.tight_layout()
plt.show()

plt.figure(figsize=(10, 6))
residual_norms = [np.sqrt(np.mean(r**2)) for r in residuals]
plt.plot(residual_norms, 'b-', linewidth=2)
plt.xlabel('Boosting Iteration')
plt.ylabel('RMSE (Residual)')
plt.title('Gradient Boosting: 잔차가 감소하는 과정')
plt.grid(True, alpha=0.3)
plt.show()

과적합 제어 전략

1. Learning Rate (η)

Shrinkage: 각 트리의 기여도를 줄임

  • 작은 learning_rate + 많은 n_estimators = 좋은 성능
  • 일반적으로 0.01 ~ 0.1 사용
  • Trade-off: 성능 vs 학습 시간

2. Tree 제약

  • max_depth: 개별 트리 깊이 제한 (3~8)
  • min_samples_split: 노드 분할 최소 샘플
  • min_samples_leaf: 리프 노드 최소 샘플
  • max_features: 특성 샘플링

3. Subsampling

Stochastic Gradient Boosting

  • subsample < 1.0: 각 트리마다 데이터 샘플링
  • 과적합 감소 + 학습 속도 향상
  • 일반적으로 0.5 ~ 0.8

4. Early Stopping

검증 손실이 개선되지 않으면 학습 중단

  • validation_fraction: 검증 데이터 비율
  • n_iter_no_change: 개선 없이 지속할 라운드
  • tol: 개선 판단 임계값

XGBoost: Extreme Gradient Boosting

복습: GBM의 특징

  • AdaBoost와의 차이: 가중치를 직접 지정하지 않음
  • 자동 가중치: 데이터별로 gradient(기울기) 값이 상이함
  • 핵심 메커니즘: gradient 값이 자동으로 가중치 역할
오차가 큰 데이터 = gradient 큼 = 더 중요하게 학습

GBM(Gradient Boosting)의 한계

  • 느린 학습 속도: 순차적 처리만 가능 (병렬화 어려움)
  • 단순한 분할 기준: 1차 미분(gradient)만 사용
  • 과적합 위험: 정규화 기능 부족
  • 메모리 비효율: 대용량 데이터 처리 시 느림
  • 결측치 처리: 사전 전처리 필수

XGBoost = GBM + 5가지 혁신

"Gradient Boosting을 극한(Extreme)까지 최적화!"

  1. 병렬 처리: 노드 분할 기준을 동시에 탐색 → 10배 빠른 속도
  2. 2차 미분 활용: 더 정교하고 안정적인 학습
  3. 강력한 정규화: 과적합 자동 방지
  4. 결측치 자동 처리: 전처리 불필요
  5. 스마트 가지치기: 효율적인 트리 구조

한 줄 요약

XGBoost = GBM + 터보 엔진(병렬) + 정밀 센서(2차 미분) + 자동 조종(정규화)

GBM이 경차라면, XGBoost는 최신 스포츠카!

XGBoost 혁신 ① - 병렬 처리로 속도 향상

GBM: 순차적 처리

비유: 요리하기 (혼자서)

  • 재료 1 자르기 → 완료 후 → 재료 2 자르기 → 완료 후 → 재료 3 자르기
  • 한 번에 하나씩만 처리
  • 특성(feature)을 하나씩 순차적으로 확인
for feature in features:
    # 이 특성으로 분할하면 얼마나 좋아질까?
    gain = calculate_gain(feature)  # 하나씩 순차 처리
    if gain > best_gain:
        best_feature = feature

XGBoost: 병렬 처리

비유: 요리하기 (팀워크)

  • 재료 1, 2, 3을 동시에 여러 사람이 자르기
  • 모든 특성을 여러 CPU가 동시에 확인
  • 노드 분할 기준을 병렬적으로 탐색
parallel_for feature in features:  # 동시에 여러 개 처리!
    gain = calculate_gain(feature)
    update_best_feature(feature, gain)

작업 GBM XGBoost 속도 향상

핵심 포인트

병렬 처리 대상: 트리 생성 자체는 순차적, 하지만 각 노드에서 최적 분할점 찾기를 병렬로 처리

결과: 데이터가 클수록 속도 차이가 더 극명하게 나타남

XGBoost 혁신 ② - 2차 미분으로 정교한 학습

수학적 비교

GBM: 1차 미분(Gradient)만 사용

손실 함수 근사: L ≈ L₀ + g·Δy

g = ∂L/∂y (1차 미분)
  • g (gradient): "현재 방향" 정보만 제공
  • 의미: "이쪽으로 가야 해!"

XGBoost: 1차 + 2차 미분(Hessian) 사용

손실 함수 근사: L ≈ L₀ + g·Δy + ½h·(Δy)²

g = ∂L/∂y (1차 미분)
h = ∂²L/∂y² (2차 미분, Hessian)
  • g (gradient): "방향" 정보
  • h (hessian): "곡률" 정보 (얼마나 빠르게 방향이 바뀌는가)
  • 의미: "이쪽으로 가되, 조심스럽게!"

비유: 네비게이션

GBM (1차 미분만 사용)

  • 정보: "지금 어느 방향으로 가야 하는가?"
  • 비유: 현재 속도만 보고 운전
  • 문제점: "앞으로 가!" 라고만 알려줌 → 커브에서 사고 위험

XGBoost (1차 + 2차 미분 사용)

  • 정보: "어느 방향으로?" + "얼마나 빠르게 방향이 바뀌는가?"
  • 비유: 속도와 가속도를 동시에 확인
  • 장점: "앞으로 가되, 곧 커브니까 속도 줄여!" → 안전 운전

구체적 예시

데이터 포인트: 실제값 = 100, 현재 예측 = 80

GBM 방식

  • 오차 = 20
  • 1차 미분(g) = 20
  • 결정: "20만큼 더 늘려!"
  • 다음 예측 = 80 + 20 = 100
  • 너무 과감하게 조정 → 불안정

XGBoost 방식

  • 오차 = 20
  • 1차 미분(g) = 20
  • 2차 미분(h) = 2 (곡률 정보)
  • 결정: "20만큼 늘리되, 곡률이 2니까..."
  • 최적 증가량 = g/h = 20/2 = 10
  • 다음 예측 = 80 + 10 = 90
  • 조심스럽게 조정 → 안정적

왜 2차 미분이 중요한가?

비유: 공을 던져서 목표 지점에 맞추기

  • 1차 미분만 (GBM): "목표가 오른쪽에 있으니 오른쪽으로 던져!" → 너무 세게 던지면 지나침
  • 1차+2차 미분 (XGBoost): "목표가 오른쪽에 있고, 곡선이 급격히 꺾이니까 적당한 힘으로!" → 정확히 도달

결과: 더 빠르고 안정적인 수렴, 과적합 방지, 높은 정확도

XGBoost 혁신 ③④⑤ - 정규화, 결측치, 가지치기

③ 강력한 정규화 (Regularization)

비유: 학생 평가

  • GBM: 시험 점수만 봄 → 과도하게 암기한 학생이 높은 점수 (과적합)
  • XGBoost: 시험 점수 + 이해도 + 응용력 종합 평가 → 진짜 실력 (일반화)

정규화 수식

목적 함수 = Σ(손실) + Ω(모델 복잡도)

Ω = γ·T + ½λ·Σw²

T = 리프 노드 수, w = 리프 가중치
  • γ (gamma): 트리가 너무 복잡하면 패널티
  • λ (lambda): 가중치가 너무 크면 패널티
  • 효과: 자동으로 "이 정도면 충분해!" 판단

④ 결측치 자동 처리

비유: 설문조사

  • GBM: 설문 답변에 빈 칸이 있으면 → 에러 발생 → 사전 전처리 필수
  • XGBoost: 빈 칸이 있어도 자동으로 "왼쪽/오른쪽 어디로 보낼지" 학습

작동 방식

  1. 노드 분할 시, 결측치를 왼쪽으로 보냈을 때 이득 계산
  2. 결측치를 오른쪽으로 보냈을 때 이득 계산
  3. 더 좋은 방향으로 자동 할당!

전처리 불필요 → 데이터 과학자의 시간 절약!

⑤ 스마트 가지치기 (Pruning)

비유: 나무 관리

GBM (Pre-pruning)

  • "최대 깊이 5까지만!"
  • 5층까지 자라다가 강제 멈춤
  • 더 자라면 좋을 수도 있는데 미리 차단

XGBoost (Post-pruning)

  • 일단 끝까지 자라게 둠
  • 다 자란 후 불필요한 가지 제거
  • "이 가지는 실제로 도움 안 되네!" → 제거
  • 더 효율적인 트리 구조

# DMatrix 생성 (XGBoost 전용 데이터 구조 - 메모리 효율적)

AdaBoost 하이퍼파라미터 튜닝 전략

주요 하이퍼파라미터

1. n_estimators (약한 학습기 개수)

의미: 순차적으로 학습할 Decision Tree의 개수

Trade-off:

  • 적을 때 (10~50): 빠른 학습, 과소적합 위험
  • 많을 때 (100~500): 높은 정확도, 느린 학습, 과적합 가능

권장 시작값: 50

탐색 범위: [10, 50, 100, 200, 500]

2. learning_rate (학습률)

의미: 각 약한 학습기의 기여도를 조절하는 가중치

Trade-off:

  • 높을 때 (1.0): 빠른 수렴, 과적합 위험
  • 낮을 때 (0.1~0.5): 안정적 학습, 더 많은 트리 필요

권장 시작값: 1.0

탐색 범위: [0.5, 1.0, 1.5, 2.0]

3. base_estimator (기본 학습기)

의미: 약한 학습기로 사용할 모델

권장: DecisionTreeClassifier(max_depth=1) - Decision Stump

탐색 범위: max_depth [1, 2, 3]

튜닝 전략

  1. 1단계: base_estimator는 max_depth=1로 고정
  2. 2단계: n_estimators 튜닝 (50, 100, 200 시도)
  3. 3단계: learning_rate 튜닝 (1.0, 0.8, 0.5 시도)
  4. 4단계: 최적 조합 찾기 (n_estimators와 learning_rate)

황금 규칙: learning_rate 낮추면 n_estimators 늘리기

AdaBoost 튜닝 예제

from sklearn.ensemble import AdaBoostClassifier

Gradient Boosting 하이퍼파라미터 튜닝 전략

주요 하이퍼파라미터

1. n_estimators (트리 개수)

Trade-off:

  • 적을 때 (50~100): 빠른 학습, 과소적합
  • 많을 때 (500+): 높은 정확도, 느린 학습

권장 시작값: 100

탐색 범위: [100, 200, 500]

2. learning_rate (학습률)

Trade-off:

  • 높을 때 (0.1~0.3): 빠른 수렴, 불안정
  • 낮을 때 (0.01~0.05): 안정적, 느린 수렴

권장 시작값: 0.1

탐색 범위: [0.01, 0.05, 0.1]

3. max_depth (트리 깊이)

Trade-off:

  • 얕을 때 (2~3): 단순 모델, 빠른 학습
  • 깊을 때 (5~7): 복잡한 패턴 포착, 과적합 위험

권장 시작값: 3

탐색 범위: [3, 5, 7]

4. subsample (샘플링 비율)

의미: 각 트리 학습 시 사용할 데이터 비율

권장 시작값: 0.8

탐색 범위: [0.7, 0.8, 0.9, 1.0]

5. min_samples_split / min_samples_leaf

의미: 노드 분할 최소 샘플 수

권장 시작값: split=2, leaf=1

탐색 범위: split [2, 5, 10], leaf [1, 2, 4]

Gradient Boosting 튜닝 전략 (계속)

단계별 튜닝 전략

  1. 1단계: 기본 트리 개수 설정
    • n_estimators = 100 (임시 고정)
    • learning_rate = 0.1 (임시 고정)
  2. 2단계: 트리 구조 튜닝
    • max_depth: [3, 5, 7]
    • min_samples_split: [2, 5, 10]
    • min_samples_leaf: [1, 2, 4]
  3. 3단계: 샘플링 튜닝
    • subsample: [0.7, 0.8, 0.9]
    • max_features: ['sqrt', 'log2', None]
  4. 4단계: learning_rate와 n_estimators 동시 튜닝
    • (lr=0.1, n=100), (lr=0.05, n=200), (lr=0.01, n=500)

Gradient Boosting 튜닝 예제

from sklearn.model_selection import GridSearchCV

# 1단계: 트리 구조 튜닝

황금 규칙

  • 과적합 징후: learning_rate ↓, max_depth ↓, subsample ↓
  • 과소적합 징후: n_estimators ↑, max_depth ↑
  • 균형: learning_rate ↓ ⟺ n_estimators ↑

XGBoost 하이퍼파라미터 튜닝 전략

주요 하이퍼파라미터

1. n_estimators (트리 개수)

권장 시작값: 100

탐색 범위: [100, 200, 500] (Early Stopping 사용)

2. learning_rate (eta)

권장 시작값: 0.1

탐색 범위: [0.01, 0.05, 0.1, 0.3]

3. max_depth (트리 깊이)

권장 시작값: 5

탐색 범위: [3, 5, 7, 9]

4. subsample (행 샘플링)

권장 시작값: 0.8

탐색 범위: [0.7, 0.8, 0.9]

5. colsample_bytree (열 샘플링)

의미: 각 트리 생성 시 사용할 특성 비율

권장 시작값: 0.8

탐색 범위: [0.7, 0.8, 0.9]

6. gamma (분할 최소 손실 감소)

의미: 노드 분할에 필요한 최소 손실 감소량

권장 시작값: 0

탐색 범위: [0, 1, 3, 5] (과적합 시 증가)

7. reg_alpha (L1), reg_lambda (L2)

의미: 가중치 정규화

권장 시작값: alpha=0, lambda=1

탐색 범위: alpha [0, 0.1, 1], lambda [1, 5, 10]

XGBoost 튜닝 전략 (계속)

단계별 튜닝 전략

  1. 1단계: 기본 설정으로 시작
    • n_estimators=100, learning_rate=0.1, max_depth=5
    • Early Stopping으로 최적 반복 횟수 확인
  2. 2단계: 트리 구조 튜닝

    • min_child_weight: [1, 3, 5]
  3. 3단계: 샘플링 튜닝
    • subsample: [0.7, 0.8, 0.9]
    • colsample_bytree: [0.7, 0.8, 0.9]
  4. 4단계: 정규화 튜닝 (과적합 시)
    • gamma: [0, 1, 3]
    • reg_alpha: [0, 0.1, 1]
    • reg_lambda: [1, 5, 10]
  5. 5단계: learning_rate 최적화
    • learning_rate 낮추고 n_estimators 증가
    • (lr=0.05, n=200), (lr=0.01, n=500)

import xgboost as xgb

    'max_depth': [3, 5, 7],
    'min_child_weight': [1, 3, 5]
}

    xgb.XGBClassifier(n_estimators=100, learning_rate=0.1, random_state=42),
    param_grid_step1,
    cv=5,
    scoring='accuracy',
    n_jobs=-1
)
xgb_grid1.fit(X_train, y_train)
print("1단계 최적:", xgb_grid1.best_params_)

# 2단계: 샘플링 튜닝

    'colsample_bytree': [0.7, 0.8, 0.9]

모델 선택 가이드

AdaBoost를 선택하는 경우

  • 간단한 이진 분류 문제
  • 빠른 프로토타이핑이 필요할 때
  • 약한 학습기의 성능이 50%보다 약간 좋을 때
  • 튜닝할 시간이 부족할 때

Gradient Boosting을 선택하는 경우

  • 회귀 문제 또는 다중 분류 문제
  • 특성 중요도 분석이 중요할 때
  • 모델 해석이 필요할 때
  • 중소 규모 데이터셋 (10만 행 이하)

XGBoost를 선택하는 경우

  • 대규모 데이터셋 (10만 행 이상)
  • 최고 성능이 필요할 때 (Kaggle 경진대회)
  • 결측치가 많은 데이터
  • 학습 속도가 중요할 때
  • 프로덕션 환경 (안정성과 속도 모두 필요)

최종 팁

시작은 간단하게: AdaBoost나 기본 설정의 GradientBoosting으로 베이스라인 설정

성능이 필요하면: XGBoost로 전환하고 체계적으로 튜닝

과적합 주의: 검증 세트 성능을 항상 모니터링

시간 투자: 튜닝에 시간을 많이 쓰는 것보다 feature engineering이 더 효과적일 수 있음

핵심 요약

Boosting의 핵심

  • 순차적 학습: 이전 모델의 실수를 다음 모델이 보완
  • 약한 학습기 → 강한 학습기: 여러 단순 모델의 조합
  • 편향 감소: Bagging과 달리 편향을 줄이는 효과

AdaBoost

  • 샘플 가중치 업데이트 방식
  • 오분류 샘플에 집중
  • 이진 분류에 효과적

Gradient Boosting

  • 잔차(Residual) 학습
  • 경사하강법 원리 적용
  • 다양한 손실 함수 사용 가능

과적합 제어

  • Learning Rate: 작을수록 안정적
  • Tree Constraints: 깊이, 노드 제한
  • Subsampling: Stochastic Gradient Boosting
  • Early Stopping: 최적 라운드 찾기
